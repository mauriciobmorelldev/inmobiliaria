# Presentacion Comercial - Plataforma Inmobiliaria Connexa

## 1. Idea Principal

Connexa es una plataforma inmobiliaria web lista para adaptar a distintas inmobiliarias. Combina sitio publico, catalogo de propiedades, fichas completas, area de clientes y panel administrativo en una sola solucion.

La propuesta no es solo tener una pagina web, sino una herramienta operativa para captar consultas, ordenar propiedades, administrar colaboradores y mejorar la experiencia del cliente final.

## 2. Problema Que Resuelve

Muchas inmobiliarias trabajan con informacion dispersa:

- Propiedades cargadas en planillas o sistemas separados.
- Consultas que llegan por WhatsApp, mail o redes sin trazabilidad.
- Sitios web desactualizados.
- Fichas de propiedades poco claras o poco atractivas.
- Dificultad para que cada colaborador gestione solo su cartera.
- Falta de metricas simples para entender que propiedades generan interes.
- Clientes sin un espacio propio para guardar favoritos o revisar consultas.

Connexa centraliza estos puntos en una plataforma unica.

## 3. Solucion Entregada

Se desarrollo una plataforma inmobiliaria con:

- Sitio publico responsive.
- Home moderna y editable.
- Catalogo de propiedades con filtros.
- Fichas completas con galeria y zoom.
- Formulario de consulta por propiedad.
- Botones de compra y alquiler.
- Area de cliente final.
- Panel administrador.
- Gestion de propiedades.
- Gestion de colaboradores.
- Gestion de clientes.
- Gestion de leads.
- Estadisticas operativas.
- Branding configurable.
- Base preparada para integracion con sistemas externos como Tocco.

## 4. Valor Para Una Inmobiliaria

### Mejor experiencia para el cliente

El cliente puede navegar propiedades, filtrar, abrir fichas completas, ver imagenes, consultar y guardar favoritos desde cualquier dispositivo.

### Mas consultas ordenadas

Cada consulta queda asociada a una propiedad, cliente y estado. Esto evita perder oportunidades comerciales.

### Gestion interna mas clara

El administrador puede cargar propiedades, editar estados, gestionar colaboradores y revisar actividad desde un panel unico.

### Control por roles

Un colaborador puede cargar y modificar solo sus propiedades. Esto permite sumar equipos comerciales sin perder control del inventario.

### Marca editable

La inmobiliaria puede adaptar textos, colores, banners y contenido de home sin tocar codigo.

### Producto escalable

La misma base se puede vender a otras inmobiliarias cambiando marca, contenido, usuarios, propiedades y configuraciones.

## 5. Modulos De La Plataforma

### Modulo 1 - Sitio Publico

Incluye:

- Home institucional orientada a conversion.
- Header responsive.
- Navegacion clara para comprar y alquilar.
- Catalogo de propiedades.
- Filtros simples e intuitivos.
- Cards de propiedades clickeables.
- Diseno mobile, tablet y desktop.

Beneficio:

La inmobiliaria tiene una vidriera digital profesional, facil de navegar y enfocada en generar consultas.

### Modulo 2 - Ficha De Propiedad

Incluye:

- Galeria de imagenes.
- Visualizador ampliado.
- Zoom por click.
- Informacion principal de la unidad.
- Caracteristicas con iconos.
- Descripcion.
- Formulario de consulta.
- Accion por WhatsApp cuando corresponde.

Beneficio:

Cada propiedad se presenta como un producto comercial completo, no como una ficha estatica basica.

### Modulo 3 - Panel Administrador

Incluye:

- Dashboard principal.
- Gestion de propiedades.
- Gestion de agentes.
- Gestion de administradores.
- Gestion de clientes.
- Gestion de leads.
- Gestion de filtros.
- Gestion de branding y home.

Beneficio:

La inmobiliaria puede operar la plataforma sin depender de un desarrollador para cada cambio cotidiano.

### Modulo 4 - Home Editable

Incluye:

- Textos principales.
- Botones de accion.
- Banners.
- Imagenes.
- Titulos de secciones.
- Paleta visual.
- Logo y nombre comercial.

Beneficio:

Permite adaptar campanas, promociones, cambios de marca o mensajes comerciales desde el panel.

### Modulo 5 - Roles Y Colaboradores

Incluye:

- Administrador principal con acceso total.
- Colaborador con permisos limitados.
- Propiedades asociadas al usuario que las creo.
- Restriccion para que un colaborador no modifique propiedades ajenas.

Beneficio:

La inmobiliaria puede sumar vendedores, agentes o captadores manteniendo seguridad y orden.

### Modulo 6 - Clientes Finales

Incluye:

- Registro.
- Login.
- Confirmacion de email.
- Perfil.
- Favoritos.
- Historial de consultas.

Beneficio:

El cliente tiene una experiencia mas moderna y personalizada, aumentando la probabilidad de volver al sitio y avanzar en una decision.

### Modulo 7 - Leads Y Seguimiento Comercial

Incluye:

- Registro de consultas.
- Estado del lead.
- Asociacion con propiedad.
- Asociacion con cliente.
- Historial de eventos.

Estados:

- Nuevo.
- Visita.
- Reservado.
- Cerrado.

Beneficio:

El equipo comercial puede seguir oportunidades y medir avance real, no solo recibir mensajes aislados.

### Modulo 8 - Estadisticas Operativas

Incluye:

- Propiedades totales.
- Propiedades disponibles, pausadas, reservadas y vendidas.
- Leads por estado.
- Conversion a visita, reserva y cierre.
- Rendimiento por agente.
- Propiedades con mas actividad.
- Evolucion de leads.

Beneficio:

La inmobiliaria puede tomar decisiones con informacion interna simple y accionable.

### Modulo 9 - Integracion Tocco

Incluye:

- Adaptador preparado para sincronizar propiedades.
- Variables de entorno.
- Logs de sincronizacion.
- Modo mock si no hay credenciales.

Beneficio:

La plataforma queda preparada para conectarse con sistemas externos y evitar doble carga cuando exista documentacion/API disponible.

## 6. Diferenciales Comerciales

- No es una landing: es una plataforma operativa.
- El administrador puede gestionar contenido y propiedades.
- El cliente final tiene cuenta propia.
- Los colaboradores tienen permisos limitados.
- La home es editable.
- La experiencia mobile fue trabajada especificamente.
- Las propiedades tienen fichas visuales con galeria y zoom.
- Se puede adaptar a distintas marcas inmobiliarias.
- Permite evolucionar hacia CRM, pagos o integraciones futuras.

## 11. Pendientes Para Version Productiva Final

Antes de venderla como SaaS o instalarla en produccion para multiples clientes, se recomienda:

- Migrar autenticacion a Supabase Auth o sesiones seguras HTTP-only.
- Hashear contrasenas.
- Activar Row Level Security en Supabase.
- Persistir confirmaciones de email directamente en base de datos.
- Guardar imagenes en Supabase Storage o CDN.
- Configurar dominio, hosting y variables productivas.
- Cambiar credenciales por defecto.
- Agregar rate limiting.
- Agregar backups.
- Crear plantilla de onboarding para nuevas inmobiliarias.
